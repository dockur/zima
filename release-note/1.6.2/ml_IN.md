## [1.6.2]

### Security
- അക്കൗണ്ട് മാനേജ്മെന്റ് മോഡ്യൂളിലെ ഉയർന്ന അപകടസാധ്യതയുള്ള ഒരു സുരക്ഷാ പ്രശ്നം അടിയന്തരമായി പരിഹരിച്ചു, അഡ്മിനിസ്ട്രേറ്റര്‍ പ്രവര്‍ത്തനങ്ങളുടെ സുരക്ഷ കൂടുതല്‍ മെച്ചപ്പെടുത്തി
- ഫയല്‍ സേവനത്തിലെ ഉയർന്ന അപകടസാധ്യതയുള്ള ഒരു സുരക്ഷാ പ്രശ്നം അടിയന്തരമായി പരിഹരിച്ചു, ഫയല്‍ ആക്‌സസിന്റെയും പ്രവര്‍ത്തനങ്ങളുടെയും സുരക്ഷ മെച്ചപ്പെടുത്തി
- സന്ദേശ സേവനത്തിലെ ഉയർന്ന അപകടസാധ്യതയുള്ള ഒരു സുരക്ഷാ പ്രശ്നം പരിഹരിച്ചു, സിസ്റ്റം ആശയവിനിമയ സുരക്ഷ മെച്ചപ്പെടുത്തി
- SSH ഇപ്പോള്‍ root ഉപയോക്താവിന്റെ റിമോട്ട് ലോഗിന്‍ ഡീഫോള്‍ട്ടായി പ്രവര്‍ത്തനരഹിതമാക്കുന്നു, ഉയർന്ന അധികാരമുള്ള അക്കൗണ്ടുകളുടെ പുറംലോക പ്രകടന സാധ്യത കുറയ്ക്കുന്നു
- brute-force ആക്രമണങ്ങള്‍ തടയാന്‍ SSH നിരവധി സുരക്ഷാ നയങ്ങള്‍ ചേര്‍ത്തിട്ടുണ്ട്
- NFS സേവനം ഇപ്പോള്‍ ഡീഫോള്‍ട്ടായി പ്രവര്‍ത്തനരഹിതമാണ്, അനധികൃത പ്രവേശന അപകടസാധ്യത കുറയ്ക്കുന്നു

### New
- Files ആപ്പിന് ഡാര്‍ക്ക് മോഡ് ചേര്‍ത്തു
- Alibaba Cloud Drive ഇന്റഗ്രേഷന്‍ പിന്തുണ ചേര്‍ത്തു
- Linux kernel 6.18.9 ലേക്ക് അപ്ഡേറ്റ് ചെയ്തു
- NVIDIA 470 / 580 / Open Kernel Module, Intel Xe / Battlemage, Realtek RTW88 / RTL8127, Intel / MediaTek / AIC8800 വയര്‍ലെസ് ഉപകരണങ്ങള്‍, Cirrus Logic ഓഡിയോ ഉപകരണങ്ങള്‍ എന്നിവയ്ക്കുള്ള ഡ്രൈവര്‍ പിന്തുണ ചേര്‍ത്തതോടൊപ്പം, ഹാര്‍ഡ്‌വെയറിനെ അടിസ്ഥാനമാക്കി യോജിച്ച ഡ്രൈവര്‍ പതിപ്പ് സ്വയം പൊരുത്തപ്പെടുത്തുന്ന NVIDIA ഡ്രൈവര്‍ ഓട്ടോ-ഡിറ്റക്ഷന്‍-അഡാപ്റ്റേഷന്‍ സംവിധാനവും ചേര്‍ത്തു

### Fixed
- OneDrive കീയുടെ കാലാവധി കഴിഞ്ഞതിനെ തുടര്‍ന്നുണ്ടായ കണക്ഷന്‍ പ്രശ്നം പരിഹരിച്ചു, അടിസ്ഥാന ഇന്റഗ്രേഷന്‍ ലഭ്യത പുനഃസ്ഥാപിച്ചു, കീ അപ്ഡേറ്റ് സംവിധാനം മെച്ചപ്പെടുത്തി
- Collabora Server ആപ്പ് ബാക്കപ്പുകള്‍ തടസ്സപ്പെടുത്തിയിരുന്ന പ്രശ്നം പരിഹരിച്ചു, ബാക്കപ്പ് ജോലികള്‍ ശരിയായി പ്രവര്‍ത്തിക്കുമെന്ന് ഉറപ്പാക്കി
- സ്റ്റാര്‍ട്ടപ്പില്‍ ഒന്നിലധികം USB സ്റ്റോറേജ് ഉപകരണങ്ങള്‍ക്ക് ഒരേ മൗണ്ട് പോയിന്റ് നല്‍കപ്പെടാമായിരുന്ന പ്രശ്നം പരിഹരിച്ചു
- USB മൗണ്ട് പോയിന്റുകള്‍ തമ്മില്‍ തര്‍ക്കമുണ്ടാകുമ്പോള്‍ പാരമ്പര്യ `-device-name` സഫിക്സുകള്‍ സൃഷ്ടിക്കപ്പെട്ടിരുന്ന പ്രശ്നം പരിഹരിച്ചു
- ഒരേ സീരിയല്‍ നമ്പറുള്ള ഡിസ്കുകളില്‍ നിന്ന് RAID അറേ സൃഷ്ടിക്കുമ്പോള്‍ റീബൂട്ട് കഴിഞ്ഞ് RAID തകരാറിലാകുന്ന പ്രശ്നം പരിഹരിച്ചു
